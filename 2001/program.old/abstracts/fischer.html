GMSOCKS - A Direct Socket Implementation for Myrinet 

Markus Fischer 
fischer@ti.uni-mannheim.de
 University of Mannheim

This paper describes GMSOCKS, a thin software layer that maps standard
Windows socket calls onto Myrinet. GMSOCKS enables existing
distributed applications, such as distributed databases, client server
applications, which are based on TCP/IP, to transparently benefit from
the performance of a System Area Network like Myrinet. No modification
of the source code has to be made, nor is it required to relink or
recompile the application. As our results show, the performance
improvement compared to Fast Ethernet are in the order of a magnitude.
The roundtrip latency for a given application decreases from 620 usec
to 50 usec, while the bandwidth can be boosted from 7 MB/s to 101
MB/s. As a key improvement to existing TCP/IP drivers over SANs, the
new technique cuts down the overburdened TCP/IP protocol to a small
buffer management implementation, a reliable low level API assumed.
Roughly, the GMSOCKS implementation achieves 74 per cent of the raw
performance of the low level SAN API.
