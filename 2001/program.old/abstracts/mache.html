A Comparative Study of Parallel File Systems for PC Clusters, IBM SP and ASCI Red  

Jens Mache 
jmache@lclark.edu
Sharad Garg
Lewis and Clark College

With the recent popularity of cluster computing, the demand for
advanced system software for PC clusters has grown as well. Whereas
some system software components, like message passing systems, have
been in place for a long time, parallel file systems seem to just be
coming to the forefront of attention.

Parallel file systems provide high performance disk access, which is
crucial for many scientific and commercial applications. In this paper,
we evaluate the current state of the art of parallel file systems for
PC clusters. To do so, we describe three parallel file systems and
compare their performance. The three file systems are one open-source
system, PVFS, for Linux clusters and two commercial systems, IBM GPFS
and Intel PFS, for IBM SP and ASCI Red, respectively.

Our study found many strengths, weaknesses and sensitivities of these
parallel file systems. While there is still room for improvements, high
performance disk access on PC clusters is becoming a reality. Parallel
file systems for PC clusters have come a long way. Three and a half
years after I/O throughputs of more than 1 Gigabyte/sec have been
achieved for the first time (with Intel PFS on ASCI Red), PVFS can
deliver this level of I/O performance on PC clusters.
