Cluster Rolling Upgrade Using Multiple Version Support  

Ellard T. Roush
ellard.roush@eng.sun.com
Sun Microsystems, Inc.

The paper presents a design that supports rolling upgrades in a cluster
operating system. The mechanism supports multiple, concurrently active
versions of software components. This approach supports complex
software changes, where the relationships between new and old
components are not just "1-1", but also can be "many-to-one" and
"one-to-many".
